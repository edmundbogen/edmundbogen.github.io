Most people try to write a book in a chat window. It falls apart around 20,000 words — because chat has no memory of your manuscript, can’t edit files, and can’t hold your voice across chapters.

Claude Code is different. Part 1 — The One-Time Setup

You’ll do this once. It takes about an hour. After this, every writing session starts in 10 seconds.

Step 01 15 min

Install Claude Code

Go to claude.com/claude-code and follow the install instructions for your operating system (Mac, Windows, or Linux). You’ll also need a Claude Pro or Max subscription ($20–$200/month).

When install finishes, open your Terminal (Mac) or Command Prompt (Windows) and type claude. If you see a welcome screen, you’re in.

Don’t Panic

If you’ve never opened a terminal before, that’s fine. You’ll type about five commands the entire time you write this book. Claude Code does the rest.

Step 02 10 min

Create a GitHub repository for your book

GitHub is where your manuscript lives. Every edit is versioned — you can never lose a paragraph. Create a free account at github.com, then create a new private repository named after your book (e.g. my-book-title).

Open Claude Code in a new project folder on your computer and paste this:

Set up a new book project for me. Initialize a git repository, connect it to my private GitHub repo named “my-book-title,” and create the folder structure for a non-fiction book with chapter files.

Claude Code will ask a few questions and handle the rest.

Step 03 15 min

Create your Book Plan file

This is the most important file you will ever write. It’s where Claude Code learns your voice, your reader, and your structure. Every future session starts by reading this file.

Ask Claude Code:

Create a file called BOOK_PLAN.md. Include sections for: Book title and subtitle, One-sentence pitch, Target reader (one paragraph), My voice and tone (three adjectives plus three phrases I would never say), Chapter outline (16–20 chapters with one-sentence description each), and Disclaimers I need to include.

Fill it in. Take your time. This file is your North Star.

Step 04 5 min

Create one file per chapter

Do not write your book in one long document. Split it. Ask Claude Code:

Based on my BOOK_PLAN.md, create one markdown file for each chapter, numbered sequentially: 00_Introduction.md, 01_Chapter_01.md, 02_Chapter_02.md, and so on through the conclusion and appendices.

Now when you say “rewrite the opening of chapter three,” Claude Code edits only that file. Chapter seven stays untouched. This is how you avoid drift across a long manuscript.

Step 05 10 min

Build your assembly and conversion scripts

You need three tiny scripts. You won’t write them — Claude Code will. Just paste:

Create three Python scripts: (1) assemble.py — stitches all chapter markdown files into a single COMPLETE_MANUSCRIPT.md file in the correct order. (2) convert_to_docx.py — turns COMPLETE_MANUSCRIPT.md into a Word document I can send to a human editor. (3) build_kdp_pdf.py — turns COMPLETE_MANUSCRIPT.md into a print-ready 6x9 PDF for Amazon KDP upload. Set up all dependencies.

These three scripts turn your markdown files into every format you need. Run them after every round of edits. Each one takes under five seconds.

Part 2 — The Writing Workflow

Here’s where the magic happens. Stay disciplined on these five habits and your first draft will be done in 6–10 weeks.

Step 06 Day 1

Write your strongest chapter first — all the way through

Pick the chapter you’re most excited about. Write it start to finish. This becomes your voice anchor. From now on, every time you start a new session, tell Claude Code:

Before we write anything today, read Chapter 3 from start to finish. Match that voice exactly — same sentence rhythm, same level of formality, same use of stories and examples. Then read BOOK_PLAN.md. Confirm you’re ready.

This single habit is why my book doesn’t read like AI slop. The AI learns you from your best chapter, not from the internet.

Step 07 Daily

Draft in 90-minute blocks, not marathon sessions

Here’s my actual daily schedule during writing sprints:

TimeActivity
6:00 – 7:00 AMReview yesterday’s draft, outline today’s section
7:00 – 8:30 AMWriting block 1 — first draft, 1,500–2,000 words
8:30 – 10:00 AMGym / break (essential — don’t skip)
10:00 – 11:30 AMWriting block 2 — edit, add stories, build out prompts/examples

Two focused blocks. Three hours of real writing. That’s it. You’ll produce more in this cadence than people who “write all day” on weekends.

Step 08 Weekly

Commit your work to GitHub every single session

At the end of every writing session, paste:

Commit today’s work to GitHub with a descriptive message, then push to the remote repo.

This is your insurance policy. If your laptop dies, your manuscript is safe. If you break something, you can roll back. You literally cannot lose work.

Step 09 After Draft 1

Run a multi-agent editorial review

This is the part that blows people’s minds. When your first draft is complete, you can have Claude Code spin up specialized “agents” that each review your manuscript from a different angle — like hiring four different editors at once.

Ask Claude Code:

Run a four-agent editorial review on COMPLETE_MANUSCRIPT.md. Agent 1: fact-check every statistic and claim, flag anything without a source. Agent 2: voice consistency — flag paragraphs that don’t sound like my Chapter 3 voice anchor. Agent 3: reader-friendliness — flag jargon, long paragraphs, generic AI phrasing. Agent 4: structure — flag chapters that start slow or end weak. Consolidate findings into a prioritized punch list.

I got a 35-item punch list on Prompt to Close. Items 1–29 were fixed in three days. The book got dramatically tighter — 66,500 words (V1) down to 52,701 (V7). That kind of iteration is only possible when your manuscript lives in files the AI can see and edit.

Step 10 Launch Week

Publish to Amazon KDP

When you’re ready to publish, ask Claude Code:

Run build_kdp_pdf.py to generate the print-ready 6x9 PDF. Then generate the Kindle ebook file. Give me a pre-launch checklist for Amazon KDP upload including: cover dimensions, pricing recommendations, category selection, keyword research, and the optimal pre-order window.

Upload to kdp.amazon.com. Set a 7-day pre-order window for algorithmic boost. Your book is live.

The Five Habits That Matter Most

  • Write to files, not chat. If it’s not saved in a file, it doesn’t exist.
  • One chapter = one file. Never combine, never split.
  • Rebuild after every edit. Run assemble.py every time, no exceptions.
  • Read your voice anchor before drafting new chapters. This is how you preserve your voice.
  • Commit to GitHub after every session. Your insurance policy.

The Mistake Almost Everyone Makes

They treat the AI as a ghostwriter. It’s not. It’s a research assistant, editor, and production pipeline rolled into one. Your voice, your stories, your opinions — those stay yours. The AI removes the drudgery (formatting, consistency, verification, assembly, conversion) so you can focus on the parts only you can do.

Rule of Thumb

If the AI wrote a sentence you wouldn’t say out loud to a friend, delete it. Your book’s value is you. The AI’s job is to help you sound more like yourself, faster.
